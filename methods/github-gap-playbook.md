# GitHub Gap Playbook

目标：把 GitHub 上的新能力翻译成中文用户能理解、能在闲鱼低成本验证的小服务。

## 看什么

优先看这几类信号：

- Star 快速增长：说明能力刚进入主流视野。
- Issues：用户不会安装、不会配置、不会落地的地方。
- Discussions：真实使用场景和反复出现的问题。
- README：项目承诺解决的核心任务。
- Templates / examples：最适合被包装成中文教程或代配置服务。

## 不看什么

- 只看起来酷，但没有具体输入输出的项目。
- 依赖大客户、复杂部署、长周期交付的项目。
- 明显涉及爬虫滥用、隐私数据、刷量、外挂、破解的项目。

## 翻译格式

```text
GitHub 项目：
它提供的新能力：
中文用户看不懂/不会用的地方：
能翻译成的具体需求：
闲鱼测试服务标题：
测试价：
交付物：
风险边界：
```

## 判断是否值得上闲鱼

满足 3 条以上再上架：

- 普通用户不懂 GitHub 或英文 README。
- 能翻译成截图、表格、录音、资料、网页这类输入。
- 交付物是一份清单、文案、流程图、表格、教程或配置建议。
- 首单能在 1 小时内人工完成。
- 可以先收 19.9 / 39 / 99 元测试。

## 边界

- 不倒卖代码。
- 不冒充开源项目作者。
- 不承诺自动化结果。
- 不处理未经授权的隐私数据。
- 不发布爬虫、外挂、刷量、破解、绕过平台规则的服务。

# Source Mining Playbook

目标不是抓全网数据，而是训练“从真实问题中发现 AI 小产品”的判断力。

## 每天搜索的关键词

通用问题词：

```text
有没有工具
求工具
求模板
谁能帮我
怎么批量
怎么自动
每次都要手动
太麻烦了
不会写
不知道怎么回
求话术
求清单
求总结
平替
```

## 判断一条评论是否值得记录

优先记录这 4 类：

- 求工具：用户明确说想要软件、自动化、批量处理。
- 求模板：用户愿意接受标准化输出。
- 求代办：用户已经在找人帮忙，说明有付费可能。
- 抱怨现有方法：用户觉得现在的方法慢、贵、难、复杂。

## 记录格式

每次记录不要只抄一句话。至少写：

```text
用户是谁：
他们在什么场景里：
输入是什么：
想要的输出是什么：
现在怎么手动做：
为什么 AI 现在能做：
7 天 MVP 怎么验证：
```

## 验证顺序

1. 先做人工服务，不急着开发。
2. 找 5 个目标用户，用微信或表单收输入。
3. 人工加 AI 生成结果，发回给用户。
4. 问两个问题：这个结果能不能直接用？愿不愿意下次付费？
5. 有 2 个愿意付费，再做网页或自动化。

## 合规边界

- 不绕过平台权限抓取私密数据。
- 不保存用户隐私截图，除非获得明确授权。
- 做公开内容观察时，只记录需求摘要和来源，不批量搬运原文。
- 涉及医疗、法律、金融、教育升学等高风险领域时，只做辅助整理，不承诺专业结论。

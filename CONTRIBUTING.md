# Contributing

欢迎提交 AI 小产品机会，但请不要只提交灵感。这个项目更看重真实需求信号。

## 可以提交什么

- 原始需求信号：评论区、社群、用户反馈里反复出现的求助、抱怨、求工具、求模板。
- 产品机会卡：已经整理出明确输入、输出、笨办法、MVP 和收费方式的小产品机会。
- 评分修正：如果某个机会的痛感、频率、付费可能被高估或低估，可以提交调整。
- 方法改进：更好的需求挖掘关键词、验证流程、隐私处理方式。

## 提交前检查

请确认：

- 没有复制私聊、群聊或平台用户的隐私内容。
- 没有把未经验证的假设写成确定事实。
- 机会卡有明确目标用户。
- 机会卡有明确输入和输出。
- 7 天 MVP 是一个人能做出来的范围。

## 数据修改

修改数据后运行：

```bash
npm run validate
```

这个命令会校验数据字段，并重新生成 `site/data.js`。

## 状态说明

- `seed-hypothesis`：种子假设，还没有真实证据。
- `signal-found`：至少看到 3 条相似公开信号。
- `manual-tested`：已经手工或半自动服务过用户。
- `paid-tested`：已有付费或明确付费承诺。
- `archived`：暂时不值得继续。
